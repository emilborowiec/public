Content formatting with html

HTML5 comes with many new elements for formatting page content. Here we shall explore both old and new tags to see what results they yield. Most if not all of them can be replaced with css style and layout, but that results in more code. Think of those tags as of ready-made components that do what we need while hiding the details. Using them effectively can make our source code tidier and easier to maintain.

Text formatting elements

Those elements are meant to apply quick stylying on a short section of text. Browsers implement them by applying some default CSS values to their contents, which can be overridden by any stylesheet applied by the page.

This means that most of those elements become ineffective, or yield unpredictable results if the page uses one of very popular CSS reset scripts.

Quotation elements

These elements are similar to formatting elements but also do a little bit more. They are as well typically affected by reset scripts.

q - simple quotation

Simple quotation element adds quote characters around the content


He said: <q>I'm going home</q>

Gets rendered like this:

He said: I'm going home

cite - simple citation

Simple citation element renders in italics


He said: <cite>I'm going home</cite>

Gets rendered like this:

He said: I'm going home

blockquote - bigger quote block

This one can be used for multi-line or multi-paragraph passages


And here's a sample block quote:
<blockquote cite="http://www.ponderingprogrammer.net">
    <p>
        Last two weeks didn’t give me much time for hobby programming.
    </p>
    <p>
        The citation attribute is not rendred but is used by search engines.
    </p>
</blockquote>

Yields:

And here's a sample block quote:

Last two weeks didn’t give me much time for hobby programming.

The citation attribute is not rendred but is used by search engines.
